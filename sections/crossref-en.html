Activityc back to ToC or Class ToC

IRI: https://www.w3.org/ns/prov#Activity

An Activity is a process by which an Entity is produced, modified, consumed or used.

Example 
           (a) A Computation in which a Dataset of Electronic Health Records (EHR) is de-identified is an Activity. 
           
           (b) Manual annotation of the de-identified Dataset with descriptive metadata is an Activity. 
           
           (c) Selection of a patient cohort from the annotated, de-identified Dataset is an Activity. 
           
           (d) Preparation of an Article on analytic findings from the above Dataset analysis is an Activity.
has sub-classes
Computation c, DataAcquisition c
is in domain of
associatedWith op, used op
is in range of
describes op, generatedBy op

Agentc back to ToC or Class ToC

IRI: https://www.w3.org/ns/prov#Agent

An Agent is an autonomous Entity. Persons and Organizations are Agents in EVI. Agents may be associatedWith Activities, and Entities may be attributedTo Agents.

Example 
           (a) Dr. Mary Smith is an Agent of type Person. 
           
           (b) The University of Virginia is an Agent, of type Organization. 

           (c) The bioRXiv preprint archive is an Agent of type Organization.

An Article is a (typically compound) textual document conveying a set of Claims and other Representations. Among these, an Article in EVI always conveys an assertion of its authorship in the form of a bibliographic self-reference.

EvidenceGraphc back to ToC or Class ToC

IRI: https://w3id.org/EVI#EvidenceGraph

An EvidenceGraph is a directed acyclic graph Ev = (Rn, R+, R-) where Rn is a set of Representations, R+ ∈ Rn ₓ Rn is a set of supports relations, R- ∈ Rn ₓ Rn is a set of challenge relations, and R+∩R- = ∅. EvidenceGraphs are bipolar argumentation frameworks [Cayrol & Lagasquie-Schiex, 2009] represented as RDF named graphs.

Example 
           (1) A minimal EvidenceGraph for Dr. Smith's Article on baby birthweights is EG1 a EvidenceGraph {"Preprint RXiv 20200729.1" a Article creator "Smith M" a Person} for "Preprint RXiv 20200729.1" a Article
           
           (2) A more robust EvidenceGraph for the correlation result discussed in Dr. Smith's Article might be "Post-conception age was significantly correlated with birth weight." a Claim derivedFrom Dataset2 generatedBy Computation1 using Dataset1, Software1.
           
           Note: EvidenceGraphs always have the relation evidenceFor to the Representation at their root. I.e. in Example 1 above, the EvidenceGraph EG1 is evidencefor "Preprint RXiv 20200729.1" a Article.
has super-classes
Representation c
is in domain of
evidenceFor op

Representationc back to ToC or Class ToC

IRI: https://w3id.org/EVI#Representation

A Representation is an Entity which represents another Entity. Representations are always attributedTo an Agent, their creator, contributor, influencer or to an Activity they were generatedBy.

Example 
           The Dataset of babies, birthweights, and ages used to support the conclusions of Dr. Smith's preprint Article, is a Representation. So is the Article itself, as well as the Figures showing, and Captions describing, the scatterplots and correlations, presented within the Article. The scatterplot Image and the output correlation Dataset upon which is is based, are also Representations.
           
           Because Representations always have Agents or Activities they were generatedBy, whether or not they make any textually recognizable Claims, they always have at least a minimal EvidenceGraph supporting them.
           
           When the creating Activity is not documented, e.g. the Activity of "writing", or "drawing", or "photographing", the EvidenceGraph of the Representation consists the creating Agent and itscreatorrelationship to the Representation. The quality of the evidence then depends solely upon the reputation of the authoring Agent, e.g. appointments and publication record.
           
           When the creating Activity is clearly documented and reproducible, for example as a Computation, the EvidenceGraph includes the Agent associatedWith running that Computation, the parameter set or other characteristics of the Computation, the Software on which the Computation was based, and the Dataset input to the Computation.
has super-classes
Entity c
has sub-classes
Article c, Claim c, EvidenceGraph c, Identifier c, Material c, Method c, Reference c
is in domain of
challenges op, contains op, createdBy op, directlyChallenges op, indirectlyChallenges op, represents op, supports op

directlyChallengesop back to ToC or Object Property ToC

IRI: https://w3id.org/EVI#directlyChallenges

The directlyChallenges property establishes a relation between instances of Representations A and B when A asserts that B is wrong or deficient. It is non-transitive, entailing indirectlyChallenges properties on Representations supportedBy the Representation it directlyChallenges, via a property chain rule.

Example 
           Dr. Chad Jones has created a new "improved" version of scipy.stats.pearsonr, v1.5.2x, deprecating the older v1.5.2, which according to Dr. Jones, contained a bug. Dr. Jones published a note to this effect on his blog. We can link that blog post and the new version of pearsonr to the old version via a directlyChallenges property. This entails indirectlyChallenges properties on the path to the root in the EvidenceGraph of Dr. Smith' Article.
           
           Note: A challenge whether direct or indirect, is an assertion, whose truth value is also based on some form of evidence. We do not say a Representation is false, flawed or invalid, simply because it has been challenged.

challenges op

Representation c
has range

supportsop back to ToC or Object Property ToC

IRI: https://w3id.org/EVI#supports

The supports property is a transitive relation between Representations entailed by its subproperty, directlySupports, which is non-transitive. In EvidenceGraphs, if A supports B, then the correctness (truth value) of A provides a warrant for belief in the correctness (truth value) of B.

Example 
           (1) The Person "Mary Smith" supports the Article "Preprint RXiv 20200729.1". To say this is to say that the known character, credentials and reputation of the author is a support for the truth value of their Articles.
           
           2(a) The Software scipy.stats.pearsonr in the SciPy v1.5.2 package supportsthe correlation statistic computed by Dr. Smith on her baby weight and age Dataset,
           
           2(b) The computed correlation statistic supports the scatterplot Image, and 
           
           2(c) the scatterplot Image supports the Article's conclusion.
           
           Because supports is transitive, if A supports B and B supports C, then A supports C by entailment. Therefore by entailment from transitivity, the scipy.stats.pearsonr code supports the conclusion of the Article.

has characteristics: transitive

has sub-properties
directlySupports op
has domain
Representation c
has range

This notion of event is not first-class in the data model, but it is useful for explaining its other concepts and its semantics.
